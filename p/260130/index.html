在安卓上部署ONNX模型

认真研究了项目结构，进行了整理，最后变成了这样：

项目结构：

VoiceEmbeddingExtractor.ktONNX模型推理核心
VoiceRecorder.kt音频录制与处理
AudioDecoder.kt把录制的音频进行解码
VoiceDatabase.kt声纹数据库管理
MainActivity.kt主界面逻辑

voiceRecorder采用了MediaRecorder的3gp格式和AAC编码，音质比较差，据说用AudioRecord可以用WAV和PCM编码，会比较好，但我没整明白。

之后的流程是计算STFT + Mel + Log，喂给ONNX得到结果。据说应该把前处理一起做进ONNX模型，但我不知道模型的来源，所以不会搞

最后也是如愿以偿的只能跑起来但做不到精确（任何输入的匹配结果都是85%左右的相似度）

内存&CPU占用

由于推理时间过短，采用了将推理重复100遍的方式抓数据。

用Android Debug Bridge获得声纹注册程序推理前与推理时/后内存变化：

内存类型 推理前 推理后 增长 说明
Native Heap 67MB 138MB +71MB 模型和推理缓冲区
Dalvik Heap 2.3MB 7.2MB +4.9MB Java对象增加
Graphics 51MB 52MB +1MB GPU内存变化不大
Code 17MB 19MB +2MB 代码段轻微增加
这个数据是随着推理次数增加逐渐变多的，并且推理结束后在关闭程序前没有减少。

可以看到内存变化主要是本地堆内存增加导致
