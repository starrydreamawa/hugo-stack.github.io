MiniCPM-o2.6 部署

MiniCPM-o是一个多模态模型，似乎主要在电脑和IOS上部署。
经过研究cookbook得知在IOS上部署需要先在Mac上编译，所以为啥不能编译好了直接放出来我服了

之后研究了安卓部署，需要用到llama.cpp项目。第一周直接用Termux在安卓上build，由于平台不同导致文件缺失大败而归。

第二周尝试使用NDK部署，步骤如下：

在Android上打开File-Settings-Language and Frameworks-Android SDK，在SDK tools中选择NDK(Side by side)和CMake，点击Apply安装。

NDK的安装需要走谷歌仓库，很可能出现下载失败。可以选择配置代理：打开File-Settings-Appearence and Behavior-System Settings-HTTP Proxy，在URL里填写

1

https://mirrors.aliyun.com/android.googlesource.com/

网上的很多80端口的链接都不能用了，填完可以点下面的check connection验证。

然后拉取项目：

git clone git@github.com:OpenBMB/llama.cpp.git
cd llama.cpp
git checkout minicpm-omni

#可以在sdk页面找到具体路径
$env:CMAKE_PATH = "C:\Users\你的用户名\AppData\Local\Android\Sdk\cmake\版本号\bin"
$env:ANDROID_NDK = "C:\Users\你的用户名\AppData\Local\Android\Sdk\ndk\版本号"
$env:PATH = "$env:CMAKE_PATH;$env:ANDROID_NDK;$env:PATH"

mkdir build
cd build

cmake .. `
  -G "Ninja" `
  -DCMAKE_TOOLCHAIN_FILE="$env:ANDROID_NDK/build/cmake/android.toolchain.cmake" `
  -DANDROID_ABI=arm64-v8a `
  -DANDROID_PLATFORM=android-24 `
  -DCMAKE_BUILD_TYPE=Debug `
  -DGGML_OPENMP=OFF `
  -DCMAKE_C_COMPILER="$env:ANDROID_NDK/toolchains/llvm/prebuilt/linux-x86_64/bin/clang" `
  -DCMAKE_C_FLAGS="-march=armv8-a" `
  -DCMAKE_CXX_COMPILER="$env:ANDROID_NDK/toolchains/llvm/prebuilt/linux-x86_64/bin/clang++" `
  -DCMAKE_CXX_FLAGS="-march=armv8-a" `
  -DGGML_LLAMAFILE=OFF `
  -DLLAMA_CURL=OFF 

cmake --build . --config Debug -j8
cmake --install . --prefix output --config Debug

生成output文件夹内容，然后通过adb传到手机上：

1
2
3

adb shell "mkdir /data/local/tmp/llama.cpp"
adb push output/bin /data/local/tmp/llama.cpp/bin
adb push output/lib /data/local/tmp/llama.cpp/lib

模型也传到手机上：

1
2

adb shell "mkdir /data/local/tmp/models"
adb push <模型路径> /data/local/tmp/llama.cpp/models
