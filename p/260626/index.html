模型部署

核心功能: 基于 AI 的环境声音监测应用。通过麦克风采集实时音频 → 提取音频嵌入（BEATs + CLAP）→ ONNX 投影 → Qwen2-Audio LLM（llama.cpp GGUF）进行声音事件分类。支持本地推理和远程服务器推理。

本地推理管线 (pipeline/)

这是项目的核心—一个 5 步音频处理管线：

  1. AudioPreProcessor (PCM → FloatArray): PCM 转浮点、线性+Lanczos 重采样、长度固定、RMS 归一化、静音检测
  2. BeatsEmbedder (PyTorch TorchScript): 加载 beats_traced.pt (~361MB)，提取 768 维音频嵌入
  3. ClapEmbedder (ONNX): 加载 clap_fusion_encoder_mel.onnx，手动实现 STFT+Mel 滤波器组+ONNX 推理
  4. OnnxProjector (ONNX): 加载 project_fp32.onnx，将嵌入从 768D 投影到 3584D，支持单向量和时序序列
  5. QwenEngine (llama.cpp): 封装 lib 模块的 InferenceEngine，注入音频嵌入、生成文本分类结果

新增: FFmpegAudioResampler — 基于 FFmpegKit soxr 的高质量音频重采样器，替代 AudioPreProcessor 中的线性插值重采样。

麦克风 (48kHz PCM) 
  → AudioRecorderManager → PcmRingBuffer (环形缓冲, 10s)
  → MonitorController (每 2s 取最新块)
  → InferenceManager (路由)
      ├── Local 模式: 5 步管线 → Qwen2-Audio LLM → 解析 SoundEvent 列表
      └── Remote 模式: OkHttp POST → 服务器 JSON 响应
  → MonitorActivity (日志 UI 显示)

配置/设置

  • SettingsManager (线程安全单例): 管理监听模式和服务器 IP
  • 模式: “our model” (本地 Qwen) / “Pengi” (远程) / “minicpm” (远程)
  • 默认 IP: 192.168.0.102:5000
  • DebugInferenceTool: 通过修改 MonitorController 中的 DEBUG 变量进行调试

构建注意事项

  • abibFilters 仅设置 arm64-v8a 和 x86_64（lib 模块）
  • JNI libs 使用 pickFirsts 策略处理 libc++_shared.so 冲突
  • app/src/main/assets/ 中的模型文件被 gitignore（总量 ~400MB+）
  • 仓库使用阿里云 Maven 镜像加速
  • NDK 版本: 27.0.12077973

编码规范

  • 所有 Pipeline 组件均为 Kotlin object 或 class 单例，通过 getInstance(context) 获取
  • 线程安全使用 @Volatile + synchronized 或 AtomicBoolean
  • 推理在 Dispatchers.IO 上运行，UI 回调切换到 Dispatchers.Main
  • 数据类定义在 monitor/model/ 包中
  • SharedPreferences 通过 SettingsManager 单例封装
  • 共享工具函数放在 util/ 包中，避免跨文件重复定义
  • 注释语言：中文 + 英文混合（大部分中文）
  • 编译目标：Java 17，Kotlin JVM Toolchain 17
